# Contributing to Google Cloud Skillsboost 2026

Welcome to the **GC_2026** community! We are excited to have you contribute to this repository. To maintain the quality and security of our scripts, please follow these guidelines.

## Contribution Rules

> [!IMPORTANT]
> **Mandatory Script Requirement**
> All Pull Requests (PRs) **MUST** contain a script named `jattu.sh` or integrate with the existing `jattu.sh` framework. PRs without a `jattu.sh` script will not be merged.

## Script Design & Automation Standards

To ensure a "perfect" and premium experience for users, all `jattu.sh` scripts must follow these standards:

### 1. Visual Excellence & UI
- **Branding**: Always include the **GC_2026** header and community links.
- **Colors**: Use `tput` for color-coded output:
    - `BLUE` for headers and important information.
    - `YELLOW` for progress indicators.
    - `GREEN` for success messages.
    - `RED` for critical errors.
- **Progress Visibility**: Use a `show_progress` function to keep the user informed during long-running tasks.

### 2. Full Automation
- **Zero Manual Steps**: Aim for 100% automation. This include creating uptime checks, notification channels, dashboards, and alerting policies via CLI or REST API (`curl`).
- **Auto-Detection**: Use the following logic to detect Zone and Region instead of hardcoding:
    ```bash
    export ZONE=$(gcloud compute project-info describe --format="value(commonInstanceMetadata.items[google-compute-default-zone])")
    ```
- **Email Handling**: Prompt the user for their email at the start to automate notification channel setup.

### 3. Error Handling & Idempotency
- **Handle Re-runs**: Scripts must be "Idempotent" (safe to run multiple times).
- **Check Existence**: Before creating a resource (VM, Firewall, Policy), check if it already exists to avoid "Already Exists" errors.
    ```bash
    if ! gcloud compute instances describe [NAME] &>/dev/null; then create_command; fi
    ```
- **Cleanup**: Always remove temporary files (JSON configs, etc.) after the script completes.

### 4. Technical Constraints
- **Debian 12 Support**: Always use the latest stable image (Debian 12) unless the lab strictly requires an older version.
- **REST API over Buggy CLI**: Use `curl` to interact with Monitoring/Logging APIs if the `gcloud` command is unreliable or missing options.
